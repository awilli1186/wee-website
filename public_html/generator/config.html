Configuration

Site generator configuration

The site file tells Grunt about your site. What is the path for the templates? Should the rendered output be minified? What sections are included?

Let’s walk through the available settings.

Name & Description #

These values are for your internal reference only.

"name": "Site Name",
"description": "Description of site."

Config #

This allows you to enable or disable core features of the static site generator.

"config": {
    "watch": true,
    "minify": true,
    "enhanceTypography": true,
    "removeIndex": true,
    "removeTrailingSlashes": true,
    "paths": {
        "partials": "partials",
        "templates": "templates",
        "extensions": "extensions"
    }
}

Watch

Setting this to true will make grunt watch for changes.

"watch": true

Minify

This will minify the rendered output if set to true.

"minify": true

Enhance Typography

Smart quotes and other typographic features can be enabled here.

"enhanceTypography": true

Remove Index

This will remove index.html from any injected links.

"removeIndex": true

Remove Trailing Slashes

Similar to removeIndex, this will remove trailing slashes from any injected links.

"removeTrailingSlashes": true

Paths

Here you can set the paths for partials, templates, and extensions.

"paths": {
    "partials": "partials",
    "templates": "templates",
    "extensions": "extensions"
}

Env #

This is where you will specify the URLs for your development environments.

"env": {
    "prod": {

    },
    "default": {

    }
}

Data #

Any custom data that you may need for your site can be stored here.

"data": {
    
}

Sections #

This is where you will specify the different sections of your site, and information related to each section. As the example below demonstrates, sections can be nested.

"sections": {
    "home": {
        "name": "Homepage",
        "data": {

        },
        "template": "index",
        "target": "public_html/index.html",
        "content": "content/home/body.md"
    },
    "blog": {
        "name": "Blog",
        "data": {

        },
        "template": "blog",
        "target": [
            "public_html/blog/index.html"
        ],
        "content": "content/blog/*.md",
        "sections": {
            "entries": {
                "name": "Entries",
                "data": {

                },
                "template": "entry",
                "target": [
                    "public_html/blog/content.html"
                ],
                "content": "content/blog/*.md"
            }
        }
    }
}

Name

The name for the section is set here.

"name": "Section Name"

Data

Any custom data that you may need for a section can be stored in the data key.

"data": {
    
}

Template

This is where you specify a section’s template. The html extension isn’t needed.

"template": "index"

Target

This specifies the file to which the content will be added.

"target": "public_html/index.html"

Content

The files listed here contain the content that will be added to the target file specified above. Files are compiled in the order that they are listed. Wildcards can be used as necessary.

"content": "content/*.md"
