Events →

Event binding to attach functionality to elements

Create organized interaction on your page with Wee’s simple event API. Bind events based on data attributes or as needed for any event. Includes support for mouseenter and mouseleave.

Bind #

Traverse DOM for available bindings

VariableTypeDefaultDescriptionRequired
evtsstringCurrently mapped eventsEvents object
optobject–Callback arguments
VariableTypeDefaultDescriptionRequired

Wee.events.on() should be used instead using ‘ref:id’ selectors.

Bind works the same as map except it immediately binds the provided events.

Wee.events.bind({
    'actionButton': {
        click: function() {
            // Click logic
        }
    }
});

VariableTypeDefaultDescriptionRequired

By default bound will return all bound events.

Wee.events.bound('.element');

Map #

Add bindings to bound object with optional exec object and/or init boolean

VariableTypeDefaultDescriptionRequired
evtsobject–Events object✓
aobject, booleanfalseEvent callback arguments or boolean to immediately bind the events
bbooleanfalseImmediately bind the events

Note that currently the callback arguments are only utilized when immediately binding the events. This will be resolved in a future release.

VariableTypeDefaultDescriptionRequired
<form data-bind="contactForm">
    <input type="text" name="name">
    <input type="email" name="email">
    <textarea name="message"></textarea>
    <input type="submit" value="Submit" data-bind="actionButton">
</form>

Wee.events.map({
    'contactForm': {
        init: 'contact:init',
        submit: 'contact:validateData'
    },
    'actionButton': {
        click: function() {
            // Click logic
        }
    }
}, true);

On #

Bind specified function to specified element and event

VariableTypeDefaultDescriptionRequired
selselector, element–Element to bind to✓
astring, object–Event name or object of events✓
bcallback, object–Callback for events or options object
cobject–Option parameters below
VariableTypeDefaultDescriptionRequired
onebooleanfalseRemove the event after first execution
delegateselector, element–Live watch all elements within this target
contextselector, element–Only watch elements within this context
argsarray–Callback arguments
scopeobject–Callback scope

Simple

Wee.events.on('#logo', 'click', function(e, el) {
    // Click logic
    e.preventDefault();
});

Multiple

Wee.events.on('#logo', {
    mouseenter: function() {
        // Enter logic
    },
    mouseleave: function() {
        // Leave logic
    },
    click: function(e, el) {
        // Click logic
        e.preventDefault();
    }
});

One #

Bind specified function to specified element and event for single execution

VariableTypeDefaultDescriptionRequired
selstring–Element to bind to✓
aselector, element–Event name or object of events✓
bselector, element–Callback for events or options object
carray–Option parameters, same as on
VariableTypeDefaultDescriptionRequired

Wee.events.on() should be used with the one property setting set to true.

Works exactly the same as on but immediately removes the event after the first execution.

Wee.events.one('#logo', 'click', function(e, el) {
    // Click logic
    e.preventDefault();
});

Unbind #

Remove bindings to bound object

VariableTypeDefaultDescriptionRequired
idstring–Data bound name✓
evtstring–Event name
VariableTypeDefaultDescriptionRequired

Wee.events.off() should be used instead using ‘ref:id’ selectors.

If a second argument isn’t supplied all events on the element will be unbound.

Wee.events.unbind('actionButton', 'click');
