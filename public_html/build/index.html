    Build

    Process to compile, optimize, and minify your project

    There are just a couple of tools you’ll need to compile a Wee project. We need to make a couple of assumptions about your knowledge of the command line. For instance, we’ll assume you know how to cd and run basic commands.

    Setup #

    1. Install the latest Node.js.
    2. Install the Grunt CLI using npm.
    3. Navigate to your project in the command line and run npm install to install Wee core and its dependencies.

    Tasks #

    Default

    Run node wee run from the command line to build your project once. This compiles and minifies all assets.

    node wee run

    Static Server

    Running node wee run:static will continuously compile your CSS and JavaScript changes on the fly, launch a local web server, serve the static HTML files in the specified project root, and reload as necessary.

    node wee run:static

    Local Server

    For more advanced needs, you will probably need to run your own local server to serve PHP and/or MySQL with something like MAMP (OS X) or EasyPHP (Windows). In that case you would use the node wee run:local command. This uses the proxy settings you have specified in wee.json and proxy to your local web server. As with the “static” command, Wee will continuously watch and compile your changes as you make them and reload your browser as necessary.

    node wee run:local

    Generate

    This task will generate wee.json configured static sites on demand. Read more about static generation.

    node wee run:generate

    Reset

    To remove unnecessary repo files and sample assets from a default Wee install run the reset task.

    node wee reset

    Validate

    Validate the current projects JavaScript source against defined standards. Read more about validation.

    node wee validate

    Test

    Kick off JavaScript unit tests using Intern. Read more about testing in Wee.

    node wee test

    Update

    The update task will notify you if there is a newer version of Wee available.

    node wee update

    Script Build #

    1. The primary script compiles to public/assets/js/script.min.js from the following:
      1. Core features enabled in wee.json
      2. Vendor files from source/js/build/vendor
      3. Remaining files from source/js/build
      4. Build files set in wee.json
      5. Core script from source/js/script.js
      6. Modules from source/modules
        1. Core CSS from core/js/script.js
        2. Remaining files from core/js
        3. Build files set in module.json
    2. Scripts in source/js/lib minify into public/assets/js/lib
    3. Custom wee.json directives compile to public/assets/js
    4. Module script in modules/*/core/js/lib minify into public/assets/modules/*/js/lib

    Style Build #

    1. The primary stylesheet compiles to public/assets/css/screen.min.css from the following:
      1. Wee reset
      2. Core features enabled in wee.json
      3. Vendor files from source/css/build/vendor
      4. Remaining files from source/css/build
      5. Build files set in wee.json
      6. Core CSS from source/css/screen.less and source/css/breakpoints
      7. Modules from source/modules
        1. Core CSS from core/css/screen.less and core/css/breakpoints
        2. Remaining files from core/css
        3. Build files set in module.json
    2. Stylesheets in source/css/lib minify into public/assets/css/lib
    3. Custom wee.json directives compile to public/assets/css
    4. Module CSS in modules/*/core/css/lib minify into public/assets/modules/*/css/lib
