Validation

Ensure consistent code quality and style

There are just a couple of tools you’ll need. We need to make a couple of assumptions about your knowledge of the command line. For instance, we’ll assume you know how to “cd” and run basic commands.

Task #

If watch is set to true in the validate section of your project.json then your scripts are validated as you save changes. Alternatively, if you just want to run validation on demand the Grunt task can be called from the command line.

grunt validate

Script #

Wee leverages JSHint and JSCS for automatic and manual code validation. JSHint serves as more of a high-level quality check whereas JSCS is focused on code consistency and formatting. In the project.json you can disable either independently or point them to a new path.

Style #

Wee doesn’t currently support automated CSS testing in the build process or validation task. However, Wee does ship with both CSScomb and CSSLint files in the project root that reflect the internal Wee standards. These can be used along with code editor plugins to validate your stylesheets.
